# 小説ジェネレーター (Web Novel Generator)

AIを活用した小説・文章執筆支援ツールです。キャラクター設定、世界観設定、プロットから小説を自動生成できます。

**🌐 デプロイ先**: https://web-novel-generator.vercel.app/

> **ブランチ状態**: このリポジトリの現在のコードは **プラットフォーム拡張前（投稿・閲覧・ランキング機能追加前）** の状態です。
> 今後の拡張計画については [ロードマップ](#ロードマップ今後の拡張計画) を参照してください。

---

## 特徴

### 📚 設定からキャラ・プロット・本文まで、ぜんぶここで完結

世界観設定 → キャラクター → 全体プロット → 詳細プロット → 小説本文、という創作の流れをすべて1つのツールで管理。各素材はAI生成時に自動で参照されるため、**設定とブレない一貫した文章**が生成されます。

```
世界観設定 ──┐
キャラクター ─┼──▶ 詳細プロット生成 ──▶ 小説本文生成
全体プロット ─┘
```

### ✏️ 気になる箇所だけ、指示1つで即修正

生成した文章の一部を選択して「もっとテンポよく」「ここをもっと感情的に」と指示を送るだけで、その箇所だけ複数の修正案を比較生成。全体を書き直す必要はありません。

```
【選択した文章】
  その瞬間、少女の表情が崩れた。

【修正指示】 → "もっと感情的に、動作も加えて"

┌─ 案1 ─────────────────────────────────────────┐
│ その瞬間、少女の唇が震え、こらえていた涙が    │
│ 一粒だけ頬を伝った。                          │
│                                   [この案を使う] │
├─ 案2 ─────────────────────────────────────────┤
│ その瞬間、少女は剣を取り落とした。かたかたと  │
│ 地面に転がる音だけが、森の中に響いた。        │
│                                   [この案を使う] │
├─ 案3 ─────────────────────────────────────────┤
│ その瞬間、少女が両手で顔を覆った。肩が、      │
│ 小さく小さく、震えている。                    │
│                                   [この案を使う] │
└───────────────────────────────────────────────┘
```

## 機能概要

### コンテンツ管理
| 機能 | 説明 |
|------|------|
| **キャラクター管理** | キャラクターの外見・性格・背景などを設定。AIによる自動生成も可能 |
| **世界観設定** | 物語の舞台・時代・ルールなどを設定 |
| **全体プロット** | 物語全体の流れ、各話の構成、章の設定を管理 |
| **詳細プロット** | 各話ごとの詳細なシーン構成をAI生成 |
| **あらすじ** | 物語の概要をAI生成 |
| **アイデア管理** | 着想・メモをフォルダ別に管理。番号付きテキストからの一括登録、AIによるアイデア生成も可能 |
| **小説管理** | 生成した小説エピソードをフォルダ別に管理 |
| **表現設定** | 文体・表現スタイルをまとめた「表現概要」と、参考にする「参考文」を管理。参考文からAIで表現概要を自動生成することも可能 |

### AI生成機能
- **キャラクター自動生成** - 全体プロット・詳細プロットを参照して新しいキャラクターを生成
- **詳細プロット生成** - 全体プロットとキャラクターから各話の詳細プロットを生成
- **あらすじ生成** - 全体プロットからあらすじを生成
- **小説エピソード生成** - 詳細プロットから小説本文を生成（目標文字数指定可）
- **修正機能** - 生成済みコンテンツを追加指示で修正・リテイク

### その他
- **プロジェクト管理** - 複数の作品をプロジェクト単位で管理。お気に入り登録でサイドバーに即アクセス
- **プロジェクト全体出力** - プロジェクト内の全コンテンツ（キャラクター・設定・プロット・詳細プロット・アイデア）をまとめてテキストコピー
- **ジェネレーター切り替え** - 「小説」「文章」などモードを切り替えてラベルを最適化
- **全体検索** - プロジェクトをまたいだキーワード検索
- **生成履歴** - 過去の生成履歴・トークン使用量を記録
- **Google認証** - Googleアカウントでログイン
- **日本語/英語** の多言語対応

## セットアップ

### 必要条件
- Node.js 18以上
- PostgreSQL
- OpenAI APIキー
- Google OAuth認証情報（Googleログインを使う場合）

### インストール

```bash
# リポジトリをクローン
git clone https://github.com/kanta7/Web_Novel_Generator.git
cd Web_Novel_Generator

# 依存関係をインストール
npm install

# 環境変数を設定
cp .env.example .env.local
# .env.local を編集して各値を設定
```

### 環境変数

`.env.local` に以下を設定してください：

```env
OPENAI_API_KEY=sk-proj-your-api-key-here
DATABASE_URL=postgresql://user:pass@localhost:5432/manga_db
NEXTAUTH_SECRET=your-random-secret
NEXTAUTH_URL=http://localhost:3000
GOOGLE_CLIENT_ID=your-google-client-id
GOOGLE_CLIENT_SECRET=your-google-client-secret
```

### DBセットアップ

```bash
# マイグレーション実行
npm run db:migrate

# (任意) Prisma Studio でDB確認
npm run db:studio
```

### ローカルデータをVercel（Neon DB）へ同期

`.env.local` に `NEON_DATABASE_URL` を設定したうえで実行します：

```env
# Vercel ダッシュボード → Storage → Neon → Connection Details の "Direct connection" URL
NEON_DATABASE_URL=postgresql://...
```

```bash
# ローカルの未ログインデータ（userId=null）をNeonへ同期
npm run db:sync-neon

# 特定ユーザーのデータを別のユーザーIDとしてコピーする場合
TARGET_USER_ID=<VercelのユーザーID> npm run db:sync-neon
```

### 起動

```bash
npm run dev
```

ブラウザで http://localhost:3000 を開いてください。

## 使い方・実例

「悪役令嬢サバイバル〜押し付けられた領地でも、私は笑う〜」を題材に、一連の流れを紹介します。

---

### 1. プロジェクトと世界観を作る

「新規プロジェクト」を作成したら、まず世界観を書き込みます。

```
王国カルディア。五大貴族が毎年秋の「領主会議」に集まり、
その年の不人気な苦役案件（廃鉱山の復興・難民受け入れ等）を
いかに他家に押し付けるかで腹の探り合いを繰り広げる貴族社会。
「悪役令嬢」と呼ばれる主人公エリアーナの領地は財政難で、
今年もまた誰かに最悪の案件を押し付けられようとしている。
```

---

### 2. キャラクターを作る（AI生成も可）

概要を一言入れるだけで、AIが肉付きのあるキャラクターを生成します。

```
【エリアーナ・フォン・ローゼンベルク（主人公）】
赤みがかった黒髪と琥珀色の瞳。口角が自然に上がっているせいで
「嘲笑っている」と誤解されがちな17歳。
毒舌で建前が嫌いだが、仲間と決めた相手には命を張る。
財政難の北方領地を幼い頃から支えてきた現実主義者。
それが「悪役」の評判を生んでいるが、本人は「呼びたければどうぞ」と開き直っている。

【シルヴィア・フォン・ヴァレンシア（ライバル）】
完璧な金の巻き毛と水色の瞳。微笑みを絶やさない「完璧令嬢」。
その裏では常に損得勘定が走っており、エリアーナを当初は排除すべき駒と見ていた。
だが、その実力に気づいてから、複雑な感情を抱きはじめる。
```

---

### 3. プロットを組む

全体の流れを書けば、AIが各話の詳細プロットを自動生成します。

```
【国内編・全体プロット】

第一幕：今年の領主会議に「呪われた北の廃鉱山を復興せよ」という
        史上最悪の苦役案件が降ってわいた。
        財政難のローゼンベルク領に押し付けようと、五大貴族が一斉に動き出す。
        根回しも同盟もない。味方はゼロ。
        ――それでもエリアーナは笑っていた。「面白くなってきた」と。

第二幕：最大の敵と思っていたシルヴィアが、実は同じ構造に嵌められていたと発覚。
        犬猿の仲の二人がしぶしぶ手を組むと、令嬢連合の工作がみるみる露わになる。
        口げんかしながらも息が合っていく二人に、周囲が「え、なんで仲いいの？」と
        戸惑いはじめる。

第三幕：廃鉱山を「超お得な投資案件」に見せる情報操作で逆転成功――のはずが、
        なぜかエリアーナ本人が視察同行役として現地に立っていた。
        崩れた坑道、漂う異臭、足元でなにかが動く。「……話が違う」
        隣のシルヴィアも、はじめて微笑みが完全に消えていた。
        「引き返しましょう」「同意します」――これが二人の、初めての完全な一致だった。
```

---

### 4. 本文を生成・修正する

詳細プロットと追加指示を渡すだけで、本文が生成されます。

```
【追加指示の例】
「エリアーナとシルヴィアの初対決シーン。
 シルヴィアの完璧な微笑みの裏側と、エリアーナの開き直りの強さを対比させて。
 一歩も引かないテンポの速い掛け合いで」
```

**↓ 生成された本文（冒頭）**

```
　夜会の会場は、甘い香水と腹の探り合いで満ちていた。

　「お待ちになっていた方なら、もう帰りましたわよ」
　振り向く前に、声の主はわかった。甘くて冷たい、完璧に計算された声音。
　シルヴィア・フォン・ヴァレンシアが、扇を口元に当てて微笑んでいた。

　「……先手を打ちましたか。やることが早い」
　「ローゼンベルクの令嬢って、こんなに率直なのね。噂通り」
　「悪役令嬢と呼びたければどうぞ。で、取引の条件は？」

　シルヴィアの微笑みが、ほんの一瞬だけ固まった。
```

気に入らない箇所は選択して「ここだけ修正」も可能。全体を書き直す必要はありません。

## ロードマップ（今後の拡張計画）

現在のバージョンは「個人の執筆支援ツール」ですが、今後は**読者・投稿者が集まるプラットフォーム**へと拡張する予定です。

### 1. 投稿・閲覧ページ

生成した小説を公開・閲覧できる機能を追加します。

**検討中の機能:**
- 作品の公開/非公開設定
- 読者向けの作品一覧・閲覧ページ（シリーズ管理、話数ナビゲーション）
- コメント・感想機能
- ブックマーク・続きから読む機能
- 作者プロフィールページ

**技術的な検討事項:**
- 公開フラグと閲覧権限の分離（ドラフト / 公開 / 限定公開）
- SEO対応（OGP、静的生成 or ISR）
- 長文表示の最適化（ページネーション or 無限スクロール）

### 2. ランキングページ

人気作品・急上昇作品をランキング形式で表示します。

**検討中の機能:**
- 総合ランキング・ジャンル別ランキング
- 週間/月間/累計の切り替え
- 閲覧数・ブックマーク数・コメント数による集計
- 新着作品・完結作品フィルター
- タグ・ジャンル検索との連携

**技術的な検討事項:**
- カウンターの集計方法（リアルタイム vs バッチ処理）
- キャッシュ戦略（Redis等の導入検討）
- スパム・不正閲覧対策

### 3. プロ機能の本格実装

現在はタイトル・本文の記録のみの「たたき台」として実装されています。今後、以下の7つの専門機能を本格的に開発予定です。

| 機能名 | 概要 |
|--------|------|
| **相関図** | キャラクター間の関係性をグラフ・ツリー形式で可視化 |
| **伏線・フラグ管理** | 伏線の設置箇所と回収予定を一覧管理。未回収の伏線をチェック |
| **時系列管理** | 物語内の出来事を時系列で整理。複数視点の時間軸も管理可能 |
| **専門用語・固有名詞辞書** | 作品固有の用語・地名・組織名などを辞書化。AI生成時に自動参照 |
| **パラメータ設定** | キャラクターや世界観の数値パラメータを定義・管理 |
| **分岐検討** | ストーリーの分岐パターンを比較検討。複数の展開案を並べて評価 |
| **レビュー** | 生成した小説・プロットを多角的な観点でAIにレビューさせる |

**カスタム機能追加の仕組みも実装済み**（＋ボタンで独自の機能タブを追加可能）。

### 4. AI漫画ネーム補助ページ

小説の文章からAIが漫画用のネーム（コマ割り・セリフ・ト書き）を生成する機能を追加します。

**検討中の機能:**
- 小説本文からコマ構成・セリフ・ト書きを自動生成
- コマ割りのビジュアルプレビュー（簡易レイアウト表示）
- ネームのエクスポート（テキスト / PDF）
- キャラクター設定と連携した感情表現の指示生成
- 「この場面を○コマで」などの指示入力

**技術的な検討事項:**
- ネーム専用プロンプト設計（コマ番号・視点・感情・セリフを構造化出力）
- `generatorType: "manga"` としてジェネレーター設定に追加
- 将来的なAI画像生成ツール（Stable Diffusion等）との連携

### データベース構造（PostgreSQL）

ユーザーが作成するデータはすべて PostgreSQL に保存されます。

```
[User] ─────────────────────────────────────────────
  │  Googleアカウント情報・表示名・アバター絵文字
  │
  └─▶ [Project] 作品プロジェクト
        │  name, memo, generatorType, isFavorite
        │
        ├─▶ [Character]     キャラクター設定
        │     slug, name, content
        │
        ├─▶ [Setting]       世界観設定
        │     slug, name, content
        │
        ├─▶ [Plot]          全体プロット
        │     slug, name, content
        │
        ├─▶ [Synopsis]      あらすじ
        │     slug, name, content
        │
        ├─▶ [DetailedPlot]  詳細プロット
        │     slug, name, content
        │
        ├─▶ [IdeaFolder]    アイデアフォルダ
        │     │  name, generatorType
        │     └─▶ [Idea]    アイデア
        │              slug, name, content
        │
        ├─▶ [NovelFolder]   小説フォルダ
        │     │  name
        │     └─▶ [NovelEpisode]  エピソード本文
        │              slug, name, content
        │
        ├─▶ [ExpressionSummary]  表現概要
        │     slug, name, content
        │
        ├─▶ [ExpressionReference] 参考文
        │     slug, name, content
        │
        ├─▶ [History]       生成履歴
        │     type, title, promptTokens, completionTokens, model
        │
        └─▶ [EditHistory]   編集履歴
              entityType, entitySlug, entityName
```

> `generatorType` フィールドにより、同一ユーザーの「小説モード」「文章モード」などのデータを分離して管理しています。
